    银行待沟通问题列表（招行） #

    • 商务&流程

      • 商务流程是什么样的？

      • 多久可以提供测试参数？

        2~3个工作日

      • 是否具备现成的联调环境？

        有联调环境

      • 是否有专人配合联调？

        有专人配合

      • 开发完成后银行是否需要在测试环境上验收？

        测试环境需要验收，执行案例并填写银行给的测试报告

      • 生产参数及前置机软件多久可以提供？

        2~3个工作日

      • 我方投产后行方是否需要生产环境的验收？

        我方自行验证

    • 跨行

      • 大额、小额、超网 的限额及时效各是什么样的？

        大额：实时，100W以上（5 x 8） 小额：非实时，100w以下（7 x 24） 超网：实时，100w以下（7 x 24）

      • 未走超网的跨行业务是否一定要上送开户行行名、开户所在地、开户行支行名或联行号？

        大小额需要上送；超网不需要上送

      • 他行的存折能否处理？

        原则上支持存折，具体取决于收款行

      • 跨行的退票接口哪几个字段可以到原业务批次号跟交易流水号？

        可以关联

      • 提供支持超网的银行列表

        ==待银行会后提供==

    • 业务

      • 单笔/单批/单日/单月/单年金额、笔数上限各为多少？

        有上限，具体限额根据商务协议定

      • 超网代发接口中chnflg= Y，但是金额超过5W，是否会自动走大小额？

        该笔会失败

      • 不同法人的子公司账户通过一个银企接入具体应该如何操作？

        通过关联操作可以实现子公司账户统一以集团编号对接银行

      • 能否提供全量的错误码与错误信息及对应的业务状态？

        提供不全，但只需判断业务的状态码

      • 能否提供详细的接口文档

        提供新版云直连接口

      • 批次提交后多久可以获取回盘？

        最少10秒后获取回盘

      • 节假日是否提供服务？

        提供

      • 支持哪些费用，是否有编码

        招行仅支持代发工资、代发其他两种

      • 批付业务是逐笔记账还是并笔记账？

        按照批次金额并笔记账

      • 文档中未找到对账接口，有无对账？

        无对账

    • 性能

      • 银行侧是否有流量控制？

        不超过5个并发

      • 最大支持多少并发数？

        不超过5个并发

      • 正常的接口请求平均响应时间是多少？

        秒级

    • 网络相关

      • 云直联是否通过公网接入？能否提供专线接入？

        提供公网与专线两种方式接入

    • 前置机相关

      • 单套前置机最大支持多少并发数？

        不使用前置机方案

      • 使用U-KEY还是软证书？

        不使用前置机方案，不涉及U-Key

      • 能否使用多个U-KEY部署多套前置机?

        不使用前置机方案，不涉及U-Key
