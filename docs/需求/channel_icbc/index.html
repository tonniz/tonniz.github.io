    银行待沟通问题列表（工行） #

    工行 #

    • 商务&流程

      • 商务流程是什么样的？

      • 多久可以提供测试参数？

        2~3个工作日

      • 是否具备现成的联调环境？

        ==工行近期更换环境，1月下旬到1月底才有联调环境==

      • 是否有专人配合联调？

        专人配合

      • 开发完成后银行是否需要在测试环境上验收？

        工行不验收，但需要易才出示带公章的验收报告，工行提供模板

      • 生产参数及前置机软件多久可以提供？

        2~3个工作日

      • 我方投产后行方是否需要生产环境的验收？

        不验收

    • 跨行

      • 大额、小额、超网 的限额及时效各是什么样的？

        大额：实时，100W以上（5 x 8） 小额：非实时，100w以下（7 x 24） 超网：实时，==5w以下==（7 x 24）

      • 跨行业务是否要上送开户行行名、开户所在地、开户行支行名或联行号？

        大小额、超网均需要上送开户行信息（联行号与开户行名2选1）

      • 他行的存折能否处理？

        原则上支持存折，具体取决于收款行

      • 跨行的退票如何处理？有没有接口可以获取？

        ==无接口可用，只能线下人工处理==

      • 若有退票，是否可以关联到原批次与原明细？

        不支持退票查询

    • 业务

      • 单笔/单批/单日/单月/单年金额、笔数上限各为多少？

        无限额

      • 能否提供全量的错误码与错误信息及对应的业务状态？

        提供不全，以业务状态码为准

      • 能否提供详细的接口文档

        需提供更详细的开发手册

      • <企业财务室指令提交>接口请求最大支持多少笔明细？

        1000笔

      • 批次提交后多久可以获取回盘？

        ==批量模式1个小时~1个半小时== ==单笔模式立马可以返回结果==

      • 节假日是否提供服务？

        提供

      • 批付业务是逐笔记账还是并笔记账？

        可并笔，但记账维度与我方不符

      • 文档中未找到对账接口，有无对账？

        无对账

    • 性能

      • 银行侧是否有流量控制？

        前置机有流量控制

      • 最大支持多少并发数？

        ==前置机模式不支持并发，加密机模式支持1000并发==

      • 正常的接口请求平均响应时间是多少？

        秒级

    • 网络相关

      • 通过公网接入？能否提供专线接入？

        提供专线与公网两种方式接入（加密机模式直能用专线接入）

    • 前置机相关

      • 单套前置机最大支持多少并发数？

        不支持并发

      • 使用U-KEY还是软证书？

        同事支持U-KEY与软证书

      • 工行银企的前置机能否部署在虚拟化的操作系统？

        工行不建议使用前置机，推荐加密机来实现高高并发

      • 能否使用多个U-KEY部署多套前置机?

        工行不建议多前置机，推荐加密机来实现高高并发
